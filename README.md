六、核心需求详解：为什么这样设计？

6.1 玩法追踪引擎 — 为什么需要它？

问题：投放人说「要找开箱类竞品」，当前需要在榜单里一个个游戏看介绍，逐个确认。费时且容易遗漏。

本产品的解决方案：
【传统流程】：看排名 → 点进应用商店 → 查介绍 → 推断玩法 → 人工记录  (15min)
【优化流程】：看排名 → 查分类筛选 → 开箱类榜单秒出  (30秒)

实现方式（规则 + AI 双轨）：
- 规则词典 (~80%)：游戏名称模式匹配（如名含「开箱」、「宝箱」）+ 标签关键词
- AI 补全 (~20%)：规则未命中 → 调 DeepSeek API → 根据「游戏名+官方介绍+用户评价」输出玩法标签，并记录证据链

为什么选择规则+AI，而不是 100% AI？
- 规则无需 API 调用，响应快（毫秒级），成本为零
- AI 只处理边缘案例，节省 80% 调用成本，总耗时从 5h 降至 30min
6.2 买量追踪引擎 — 为什么要集成素材库？

问题：投放人当前的工作流是「发现竞品买量 → 看 ADX 素材库找创意参考 → 在榜单里对比排名」，需要跨越两个工具。而且 ADX 账号稀缺，经常看一半断线。

解决方案：
- 把竞品素材（视频+图）直接内嵌到「游戏详情卡片」中
- 每条素材显示源数据（平台、投放时间、CTR、曝光量）
- 支持预览和授权下载
本期约束与分阶段方案：
1. P0（本周可启动）：人工导出 ADX 数据 → 上传 CSV → 平台导入解析（1-2h setup）
2. P1（2-3 周后）：若 ADX 协议允许自动化 → 实施 RPA + 定时任务
3. P2（若 P1 失败）：每周人工导出一次（运维成本 ≤2h/周）

为什么不一开始就做 100% 自动化？企业合规限制（ADX 用户协议通常限制爬取和自动登录）。先用「可立即启动的人工方案」验证价值，再争取合规升级。

6.3 变现模式识别 — 为什么要自动判断游戏是 IAA/IAP？

问题：投放定向需要考虑「这个品类玩家的付费意愿」。不同变现模式对应完全不同的受众：
- IAA（纯广告）→ 轻度, 免费玩家，走量模式，CPC 低
- IAP（纯付费）→ 重度，付费玩家，走粘性，CPA 高
- 混变 → 中度，双需求，需要精准人群
当前无自动化识别，投放靠经验猜测，或逐个查下载，效率低且易出错。

自动判断逻辑（结构化入库）：
规则（90% 覆盖）：
  - 畅销榜排名高 + 畅玩榜排名高 → 混变
  - 畅销高 + 畅玩低   → 纯付费
  - 畅销低 + 畅玩高   → 纯广告
  - 两榜都低          → 需 AI 判断

AI 补全（10%）：
  - 调 DeepSeek 解析游戏描述 + 用户评价 + 试玩笔记
  - 输出结构化 JSON：{model, confidence, evidence}
  - 记录证据链供人工审核

为什么要入库而不只是展示？
- 方便按「变现模式」筛选榜单（如「全找 IAP 重度产品」）
- 能在推广大屏中按模式展示趋势（「混变产品本月占比 35%」）
- 可聚合到买量模块做 ROI 对标（中期）
6.4 裂变假设清单 — 为什么不外爬，只做假设

问题：运营想知道「竞品为什么能通过社交分享放大声量」，但无法直接冷启动爬取所有分享数据和达人转化数据。

本产品的方案（低成本、可解释）：
输入：
  - 游戏官方描述 + 标签 + 玩法（从 games 表读取）
  - 试玩纪要（如果有，从游戏详情卡片人工输入）
  - 在榜数据（社交分享可能对应排名跃迁）

逻辑（仅基于现有数据，不外爬）：
  规则：
    - 若标签含「社交」+ 玩法含"PvP比赛" → 假设：竞技社交型裂变
    - 若在榜时长短 + 排名跃迁快 → 假设：内容驱动爆发
  
  AI 补全（DeepSeek）：
    - 输入「游戏概要 + 为什么能裂变」
    - 输出「分享机制 checklist」（30 秒出结果）
    
输出结构：
  {
    appid, channel (微信分享/抖音挂载/直播),
    hypothesis: "强社交裂变，分享好友后解锁进度",
    evidence: ["标签含PvP", "首周排名跃迁3-1"],
    confidence: 0.7,
    source: "ai_generated",
    updated_by: "system",
    updated_at: "2026-03-30"
  }

为什么不做全量外爬达人数据？ 
- 成本高：需要实时爬取抖音、微信分享详情，涉及隐私和反爬机制
- 周期长：达人数据需要等投放一段时间后才有，无法实时更新
- 合规风险：外爬抖音/微信数据面临法律风险
我们的权衡：用「低成本假设」替代「高成本准确」，让运营有「参考思路」而不是「实时数据」。运营可手动补充试玩纪要后，AI 会生成更精准的假设。
