# 🎮 小游戏榜单洞察平台 · minigame_tracker

> **用自建的三平台榜单监控 + 四大分析引擎，帮助投放部门在一个看板内完成「发现竞品 → 理解玩法 → 查看素材 → 判断投放价值」的完整闭环。**

---

## 📌 项目背景

2025 年小游戏市场规模达 535–610 亿元（同比 +34–40%），在投小游戏超过 5.1 万款，素材内卷超 5000 万组。在存量竞争时代，**数据响应时间比工具品牌更关键**。

本项目针对以下内部短板而生：

| 短板 | 痛点 | 本平台如何解决 |
|------|------|---------------|
| 账号稀缺 | ADX 单账号多部门竞争，频繁顶号 | 无并发限制，全员可同时访问 |
| 权限碎片化 | 榜单/素材/买量数据互盲 | 三大分析维度在一个平台统一可查 |
| 平台缺失 | 应用宝数据 ADX 覆盖不全 | 微信/抖音/应用宝三平台 9 榜完整覆盖 |
| 信息被动 | 竞品动作依赖销售定期推送 | 每日自动采集，实时更新 |
| 无历史沉淀 | 商业平台数据到期即失效 | 本地永久积累，支撑 30/90 天趋势对标 |

## 🔬 四大分析引擎

| 引擎 | 能做什么 | 核心价值 |
|------|---------|---------|
| **玩法追踪** | 自动识别开箱/靶心/LD/寻物等细粒度玩法；追踪流行度 | 投放选题：「开箱类游戏当前占榜多少？」秒出答案 |
| **买量追踪** | 接入 ADX / DataEye 素材库；记录素材投放时间、平台、效果 | 素材二次创作：直接看竞品高热素材前 3 秒截图 |
| **变现识别** | IAA 纯广告 vs IAP 纯付费 vs 混变；自动判断 + 证据标注 | 投放定向：不同变现模式对应不同人群包和出价策略 |
| **裂变假设** | 基于描述/标签/试玩纪要，输出「社交/内容/直播」驱动假设 | 运营参考：竞品为什么能裂变？有哪些机制值得复制？|

> **引擎实现原则**：规则词典（~80–90%覆盖）+ DeepSeek AI 补全（~10–20%边缘案例），兼顾速度与成本。

## 🃏 游戏详情卡片（模块②核心功能）

点击榜单任意游戏弹出详情卡，包含五个区块：

| 区块 | 内容 | 解答的问题 | 数据源 |
|------|------|-----------|--------|
| ① 基本信息 | 名称/开发商/官方介绍/玩法标签 | 这是什么游戏？ | 应用宝详情页 + DeepSeek 摘要 |
| ② 玩法 & 变现 | 大类/细分玩法/用户群/变现模式/证据 | 目标用户是谁？怎么赚钱？ | `gameplay_tags` + `game_monetization` |
| ③ 投放素材 | 竞品高热素材（视频+图）、CTR、曝光量 | 竞品用了什么创意？效果如何？ | `ad_creatives` + 本地视频缓存 |
| ④ 买量大盘 | 品类 7/30 日均耗、投放活跃度、TOP 平台 | 这个品类热不热？竞争烈不烈？ | DataEye API 或周期性导入 |
| ⑤ 同榜竞品 | 同平台同类榜单竞品列表（icon grid） | 同类还有谁在榜？排名变化如何？ | JOIN `games` & `rankings` |

## 🗄️ 数据库结构（SQLite · tracker.db）

```
games                  — 游戏基础信息
rankings               — 各榜单每日排名快照
snapshots              — 原始采集数据
ad_creatives           — 广告素材库
gameplay_tags          — 玩法标签（规则 + AI 标注）
game_monetization      — 变现模式（IAA / IAP / 混变）
virality_assumptions   — 裂变假设清单
```

---

## 🔧 数据采集说明

- **采集时间**：每日 11:00 自动执行
- **覆盖榜单**：微信小游戏（畅玩榜/畅销榜）、抖音小游戏（畅玩榜/畅销榜）、应用宝小游戏（多榜），共 9 个榜单
- **AI 富化**：调用 DeepSeek API 对规则未覆盖的游戏进行玩法标注和变现识别
- **本地存储**：所有数据永久保存于本地 SQLite，数据归属公司，不依赖第三方续费
